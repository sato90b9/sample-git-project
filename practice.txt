**WSL（Windows Subsystem for Linux）**

**Windows 10/11上で、Linux（Ubuntuなど）の環境をそのまま動かせるMicrosoft公式の機能**

wslは完全な仮想でない、Linux環境がWindows内に実在して動く

　・/home →　Linux専用領域（WSLの中）

　・/mnt/c →　WindowsのCドライブと直結　

　　mnt=mountの略(一時的にファイルシステムをマウントする場所)

WSL内部を見る方法1⃣

WSLで　explorer.exe .　を入力しエンター押すと開く

WSL内部を見る方法2⃣

①エクスプローラーのアドレスバーに「\\wsl$」※\\=￥￥でも〇

②Ubuntu(自分の使ってるディストリビューション)を開く

③homeディレクトリを開くと見れる

※ディストリビューション …Linuxカーネル単体では動作しないため、周辺ソフトウェアを組み合わせてパッケージ化したもの。

※カーネル=中心 …Linux OSの心臓部となる最重要ソフトウェア

基本的なコマンド形式： ①コマンド  ② オプション  ③ 引数 

　①実行するアクション

　②追加指示

　③コマンドが操作する対象

---

例）ls -l /home

ls: ディレクトリの内容をリスト表示

-l: 詳細オプション

/home: 対象となるディレクトリ

　↓実行結果

drwxr-x--- 3 tsato-542026 tsato-542026 4096 May  4

d →　ディレクトリ※ファイルなら「-」になる

rwx →　所有者は「読み・書き・実行OK」

r-x →　グループは「読み・実行OK」

--- →　その他の人は「何もできない」

3 →　ディレクトリ内の構造カウント

tsato-542026 tsato-542026  →　所有者ユーザー名 グループ名

4096 →　サイズ(バイト)

May  4 → 最終更新日

　r = read(読み取り)

　w=write(書き込み)

　x= excute(実行)

**「リスト表示」**

**ls** →　ファイル名だけをシンプルに一覧表示する

**ls -l →**　詳細表示。 l =long format（ロング形式）の略

ls -a →　隠しファイルを含めたすべてを表示。a= all の略

　他にも

　　ls -lh →　ファイルサイズが人間にとって読みやすい形式になる

　　　　　　（KB, MB, GBなど）h=human-readableの略

　　　隠しファイル含める場合→　ls -lah

　　ls -r →　名前の逆順(アルファベット順)で表示。

　　　ls -lr →　詳細付き、最新が下にくるため更新ファイル確認に〇

　　　ls -lR →　サブディレクトリ含め中身まで全部表示。

                                R = recursive（再帰的）

　※再帰的…自分自身を呼び出す処理が書かれている何かを呼び出す

**パーミッション（権限）のタイプ**

①読み取り（read）

②書き込み（write）

③実行（execute）

※以下のエンティティに対し設定される

エンティティ=データベースなどで管理・識別すべきモノやコトのひとかたまりを指す概念

　①所有者（user）:ファイルやディレクトリの所有者

　②グループ（group）：所有者が所属するグループ

　③その他（others）：上記以外

---

**「ファイルやディレクトリのパーミッションを表示」**

　ls -l ファイルおよびディレクトリ名

---

「**パーミッションを変更**」

**chmod コマンド**を使用する

　chmod = Change Modeの略

①シンボリックモード(文字の組み合わせ)で変更

u = 所有者　g = グループ　o = その他　a = 全員

+パーミッション追加　-パーミッション削除　=パーミッション設定

例）chmod u+x ファイル名

→　所有者に実行の権限を付与

②オクタルモード(数字で表現)で変更

読み取り(r) = 4　書き込み(w) = 2　実行(x) = 1

user　group　other　 の順

例)chmod 756 cytech.txt

　7 = 読み取り(4)+書き込み(2)+実行(1)

　5 = 読み取り(4)+実行(1)

　6 = 読み取り(4)+書き込み(2)

権限ナシなら→chmod 000 cytech.txt

---

「**所有者とグループを変更する**」

**chownコマンド**を使用する　　　chown = change owner の略

- 所有者を変更する
例）chown John sample.txt　→　sample.txtの所有者をJohnに変更する
- グループのみ変更する

　　例）chown :group1 sample.txt

　　※グループのみ変更の場合、グループ名前に「.」か「：」を記述

- 所有者とグループを変更する

　　例）chown user1:group1 sample.txt

- 指定ディレクトリ以下のすべてのファイルとディレクトリの所有者を変更する

　　例）chown -R user1:group1 dir1

**「プロセス管理」**プロセス＝実行中のプログラム

PID = 各プロセスに割り当てられたID

PPID = プロセスを生成した親プロセスのID　　PPID=Parent Process ID

ユーザー = プロセスを実行しているユーザー

優先度 = プロセスの実行優先度

状態 = プロセスの現在の状態(実行中、待機中、停止中など)

---

**「プロセス一覧を表示」**

ps →　現在実行中のプロセス一覧

　ps = process status の略

ps aux →　全プロセスの詳細情報

　　　　a: 　端末に関連付けられた全ユーザーのプロセスを表示。

　　　　u: 　ユーザー指向のフォーマット（詳細情報）を表示。

　　　　x:　端末に関連付けられていない（バックグラウンドの）プロセスも表示。

ps -ef →　全プロセスをフルフォーマットで表示

　　e=**e**very process（すべてのプロセス）

　　f=**f**ull-format

ps

top →　リアルタイムでシステムのプロセスとリソースの使用状況を表示

　q →　topを終了

　h →　ヘルプ表示

　k →　プロセスの終了（PIDの入力が必要）

　r →　プロセスの優先度の変更（PIDと新しい優先度の入力が必要）

**「プロセスを終了させる」**

kill = 終わらせる

　・指定したプロセスIDのプロセスを終了させる

　例）kill 1234

　・シグナルを指定する

　　-15(SIGTERM) : プロセスの通常の終了（デフォルト）

　　-9(SIGKILL):プロセスを強制終了

　例）kill -9 1234
